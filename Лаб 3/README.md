# Домашнее задание 03 — Проектирование и оптимизация реляционной БД

## Схема базы данных

База данных состоит из трёх таблиц:

- **`users`** — пользователи системы
- **`hotels`** — информация об отелях
- **`bookings`** — бронирования, связывающие пользователей и отели

#### `hotels`

| Поле          | Тип         | Ограничения                | Описание |
|---------------|-------------|----------------------------|----------|

| `name`        | VARCHAR(200)| NOT NULL                   | Название отеля |

| `updated_at`     | TIMESTAMPTZ  | DEFAULT NOW()                                  | Дата обновления |


## Ограничения

В схеме используются следующие ограничения:

- `PRIMARY KEY` — для идентификаторов таблиц;
- `UNIQUE` — для `users.login` и `users.email`;
- `NOT NULL` — для обязательных полей;
- `CHECK` — для контроля диапазона звёзд, даты выезда и стоимости;
- `FOREIGN KEY` — для обеспечения ссылочной целостности;
- `ON DELETE CASCADE` — при удалении пользователя или отеля связанные бронирования удаляются автоматически;
- триггеры `updated_at` — автоматически обновляют дату изменения записи.

## Индексы

Индексы добавлены для частых операций поиска и соединения таблиц:

- `users.login` — создаётся автоматически из-за `UNIQUE` и используется при поиске по логину;
- `idx_users_first_name_pattern` — ускоряет поиск по префиксу имени;
- `idx_users_last_name_pattern` — ускоряет поиск по префиксу фамилии;
- `idx_users_first_last` — полезен для комбинированного поиска по имени и фамилии;
- `idx_hotels_city` — ускоряет поиск отелей по городу;
- `idx_hotels_name` — ускоряет поиск и сортировку по названию отеля;
- `idx_hotels_stars` — помогает при фильтрации по количеству звёзд;
- `idx_bookings_user_id` — ускоряет выборку бронирований конкретного пользователя;
- `idx_bookings_hotel_id` — ускоряет соединение бронирований с таблицей отелей;
- `idx_bookings_user_status` — полезен для фильтрации бронирований пользователя по статусу;
- `idx_bookings_status` — ускоряет поиск бронирований по статусу;
- `idx_bookings_dates` — полезен для фильтрации по диапазону дат.

## Запуск проекта через Docker

### 1. Собрать и запустить контейнеры

```bash
docker compose up -d --build
```

### 2. Проверить запуск PostgreSQL

База данных будет доступна на порту `5432`.

Параметры подключения из `docker-compose.yaml`:

- **host:** `localhost`
- **port:** `5432`
- **database:** `hotel_db`
- **user:** `postgres`
- **password:** `postgres`

### 3. Подключиться к базе и выполнить скрипты

При старте контейнера PostgreSQL автоматически выполнит:

1. `schema.sql`;
2. `data.sql`.

Если нужно выполнить их вручную, можно использовать `psql`:

```bash
psql -h localhost -U postgres -d hotel_db -f schema.sql
psql -h localhost -U postgres -d hotel_db -f data.sql
psql -h localhost -U postgres -d hotel_db -f queries.sql
```

### 4. Запуск API

API-контейнер собирается из `Dockerfile` и подключается к PostgreSQL по переменной `DATABASE_URL`.
