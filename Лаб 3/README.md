# Система бронирования отелей – Лабораторная работа 3 (Вариант 13)

REST API сервис для системы бронирования отелей, построенный на
[userver framework](https://github.com/userver-framework/userver) и PostgreSQL.

---

## Схема базы данных

База данных состоит из четырёх таблиц:

### users
Хранит зарегистрированных пользователей.
- **id** – первичный ключ.
- **login** – уникальный, регистронезависимый, минимум 3 символа.
- **name**, **surname**, **email** – не могут быть NULL.
- **password_hash** – хеш пароля пользователя.

### sessions
Содержит сессионные токены для аутентифицированных пользователей.
- **token** – первичный ключ (сессионный токен).
- **user_id** – внешний ключ к `users(id)`.
- **login** – денормализованное поле для ускорения поиска.

### hotels
Содержит отели, доступные для бронирования.
- **id** – первичный ключ.
- **name**, **city**, **address** – обязательные поля.
- **description** – необязательное.
- **created_by_user_id** – внешний ключ к `users(id)` (пользователь, добавивший отель).
- **rating** – числовая оценка (0–10), необязательная.

### bookings
Записи о бронированиях, сделанных пользователями.
- **id** – первичный ключ.
- **user_id** – внешний ключ к `users(id)`.
- **hotel_id** – внешний ключ к `hotels(id)`.
- **check_in**, **check_out** – даты, обязательно `check_out > check_in`.
- **status** – пользовательский перечислимый тип: `active` или `cancelled`.

**Индексы** определены для:
- Логина в нижнем регистре (быстрый поиск)
- Города в нижнем регистре (быстрый поиск отелей)
- Составного (name, surname) (для поиска по маске)
- Внешних ключей
- Частичного индекса активных бронирований пользователя (самый частый запрос)

Подробный анализ см. в [`optimization.md`](optimization.md).

## API Endpoints

Все методы доступны по адресу `http://localhost:8080`.  
Методы, требующие аутентификации, должны получать сессионный токен в
заголовке `Authorization: Bearer <token>`.

### Публичные методы

| Метод | Путь                      | Описание                       |
|-------|---------------------------|---------------------------------|
| GET   | `/ping`                   | Проверка работоспособности     |
| POST  | `/api/v1/users`           | Регистрация нового пользователя|
| GET   | `/api/v1/users`           | Поиск пользователей            |
| POST  | `/api/v1/auth/login`      | Вход и получение токена        |
| GET   | `/api/v1/hotels`          | Список отелей (необ. город)    |

### Аутентифицированные методы

| Метод  | Путь                      | Описание                                  |
|--------|---------------------------|-------------------------------------------|
| POST   | `/api/v1/hotels`          | Создание нового отеля                     |
| POST   | `/api/v1/bookings`        | Создание бронирования                     |
| GET    | `/api/v1/bookings`        | Список бронирований аутентифицированного пользователя |
| DELETE | `/api/v1/bookings/{id}`   | Отмена бронирования (только владелец)     |

## Запуск

```bash
docker compose up --build -d
```

## Примеры использования

Все команды предполагают, что сервис запущен на `http://localhost:8080`.

### 1. Регистрация нового пользователя

```bash
curl -s -X POST http://localhost:8080/api/v1/users \
  -H 'Content-Type: application/json' \
  -d '{
        "login": "john_doe",
        "first_name": "John",
        "last_name": "Doe",
        "email": "john@example.com",
        "password": "secure123"
      }'
